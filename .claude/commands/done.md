将一条任务标记为完成并归档。

用法：
- /done                        — 列出所有进行中任务，交互选择
- /done <note_id>               — 直接指定任务
- /done <note_id> <完成备注>    — 直接指定任务和摘要

状态流转规则遵循「全局 Google Keep 标签体系规范 → state 标签」中的规定（归档前移除所有 state 标签）。

请执行以下步骤：

**第 1 步：确定目标笔记和用户备注**

- 若 $ARGUMENTS 为空：
  1. 调用 `list_notes(label="state_doing")` 获取所有进行中任务
  2. 若无结果，告知用户「当前没有进行中的任务」并终止
  3. 展示任务列表（编号、标题、project 标签），请用户选择
- 若 $ARGUMENTS 不为空：
  1. 解析参数，第一个空格前为 note_id，其余为可选的完成备注

用 `get_note` 获取笔记完整内容。

**第 2 步：生成完成摘要**

- 若用户提供了完成备注，直接使用
- 否则自动生成：
  - 提取正文中所有 `[YYYY-MM-DD]` 开头的进展记录
  - 有记录：汇总为 2-4 句中文摘要，突出主要工作和成果
  - 无记录：结合标题、正文和当前对话历史生成一句简单总结
- 向用户展示摘要，询问「以上完成摘要是否准确？可直接确认，或提供修改后的内容」
- 使用用户确认或修改后的内容

**第 3 步：归档**

1. 在正文末尾追加（空一行）：`[完成] <完成摘要>`，用 `update_note` 更新
2. 移除所有 state 标签（`state_todo`、`state_doing`、`state_blocking`、`state_waiting`）
3. 用 `archive_note` 归档
4. 确认结果，显示笔记标题和完成摘要
