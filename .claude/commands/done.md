将一条任务标记为完成并归档。

用法：
- /done                        — 列出所有进行中任务，交互选择，自动生成完成摘要
- /done <note_id>               — 直接归档（向后兼容）
- /done <note_id> <完成备注>    — 归档并追加完成备注（向后兼容）

状态流转规则遵循「全局 Google Keep 标签体系规范 → state 标签」中的规定（归档前移除所有 state 标签）。

请执行以下步骤：

**情况 A：$ARGUMENTS 为空（无参数）**

1. 调用 `list_notes(label="state_doing")` 获取所有进行中任务
2. 若无 doing 任务，告知用户「当前没有进行中的任务」并终止
3. 展示任务列表，格式：编号、标题、project 标签，请用户选择要完成的任务
4. 用 `get_note` 获取用户选中任务的完整笔记内容
5. **自动生成完成摘要：**
   - 提取笔记正文中所有 `[YYYY-MM-DD]` 开头的进展记录行
   - 若有进展记录：汇总成 2-4 句中文完成摘要，突出主要工作内容和成果
   - 若无进展记录：从笔记标题生成一句简单总结
   - 向用户展示生成的摘要，询问「以上完成摘要是否准确？可直接确认，或提供修改后的内容」
   - 使用用户确认或修改后的内容作为完成摘要
6. 在笔记正文末尾追加（与原有正文空一行）：
   `[完成] <完成摘要>`
   用 `update_note` 更新正文
7. 移除笔记上所有 state 标签（`state_todo`、`state_doing`、`state_blocking`、`state_waiting` 中存在的）
8. 用 `archive_note` 将笔记归档
9. 确认操作结果，显示笔记标题和完成摘要

**情况 B：$ARGUMENTS 不为空（有参数，向后兼容）**

1. 解析 $ARGUMENTS，第一个空格前为 note_id，其余为可选的完成备注
2. 用 `get_note` 获取笔记的当前信息，确认存在
3. 若有完成备注，在正文末尾追加一行（与原有正文之间空一行）：
   `[完成] <完成备注>`
   用 `update_note` 更新正文
4. 移除笔记上所有 state 标签（`state_todo`、`state_doing`、`state_blocking`、`state_waiting` 中存在的）
5. 用 `archive_note` 将笔记归档
6. 确认操作结果，显示笔记标题和完成备注（若有）
