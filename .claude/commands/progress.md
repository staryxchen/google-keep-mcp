在任务笔记正文末尾追加一条带时间戳的进展记录，结合对话历史和 git log 智能生成摘要。

用法：
- /progress <note_id>          — 自动从当前对话和 git log 生成进展摘要供确认
- /progress <note_id> <内容>   — 使用指定内容直接追加

请执行以下步骤：
1. 解析 $ARGUMENTS，第一个空格前为 note_id，其余为可选的进展内容
2. 用 `get_note` 获取笔记当前信息
3. 检查笔记的 state 标签：
   - 若笔记已归档，告知用户该任务已完成，不可追加进展，终止操作
   - 若状态为 `state_blocking`，提示「该任务当前被阻塞，追加进展是否意味着阻塞已解除？是否切换为进行中？」
     - 用户确认：先加 state_doing，再移除 state_blocking
     - 用户拒绝：仅追加进展记录，不改变状态
   - 若状态为 `state_waiting`，提示「该任务当前等待他人反馈，是否已收到反馈？是否切换为进行中？」
     - 用户确认：先加 state_doing，再移除 state_waiting
     - 用户拒绝：仅追加进展记录，不改变状态
   - 若状态为 `state_todo` 或无 state 标签，告知用户当前状态，询问是否同时将状态切换为 `state_doing`：
     - 用户确认：先加 state_doing，再移除旧 state 标签（若有）
     - 用户拒绝：仅追加进展记录，不改变状态
4. **若用户已在 $ARGUMENTS 中提供了进展内容（note_id 之后有文字），直接使用该内容，跳到步骤 7**
5. **收集上下文，生成进展摘要：**
   从以下来源提炼这段时间的工作进展：
   a. **当前对话历史**（主要来源）：回顾完整对话，识别与该任务相关的内容，包括：
      - 讨论过的方案、做出的决策
      - 完成的具体工作（写了什么代码、改了什么配置、解决了什么问题）
      - 遇到的问题和解决过程
      - 尚未完成但已有进展的部分
   b. **git log**（辅助来源）：运行 `git log --oneline -10`，若命令失败则跳过

   综合以上信息，生成 1-3 句简洁的中文进展摘要：
   - 聚焦「做了什么」和「结果是什么」，而非过程细节
   - 与笔记标题关联，确保摘要与任务相关
   - 若对话中没有与该任务相关的实质内容，如实告知并提示用户手动输入
6. 向用户展示生成的摘要，询问「以上进展摘要是否准确？可直接确认或修改」
   使用用户确认或修改后的内容
7. 在笔记正文末尾追加一行进展记录，格式：
   `[YYYY-MM-DD] <进展内容>`
   （使用当前日期，与原有正文之间空一行）
   用 `update_note` 更新正文
8. 确认操作结果，显示笔记标题和追加的内容
